# إعداد الإشعارات — توزيع الكلمة واللحن

## المكتبات المضافة في pubspec.yaml
```yaml
flutter_local_notifications: ^17.2.4
timezone: ^0.9.4
```

## ما تم إعداده تلقائياً
1. **AndroidManifest.xml** — تمت إضافة الصلاحيات:
   - `POST_NOTIFICATIONS` (Android 13+)
   - `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM`
   - `RECEIVE_BOOT_COMPLETED` — لإعادة الإشعارات بعد إعادة تشغيل الجهاز
   - Receiver خاص بـ flutter_local_notifications

2. **ScheduleProvider** — يجدول إشعار تلقائياً عند إضافة أي توزيع

## كيف يشتغل
- لما تضيف توزيع كلمة أو لحن، التطبيق يجدول إشعار محلي (local notification)
- الإشعار بيوصل للموبايل بتاع المدير اللي عنده التطبيق مفتوح
- فيه slider تحدد فيه كم ساعة قبل الحصة يجي الإشعار (من 1 لـ 72 ساعة)
- لو وقت التذكير عدى (يعني الحصة قريبة أو فاتت)، مش بيتجدول إشعار

## ملاحظة مهمة
الإشعارات المحلية دي بتوصل لصاحب الجهاز (الخادم/المدير اللي عنده التطبيق).
**مش push notifications** تروح لشخص تاني على موبايل تاني.

لو عايز إشعار يوصل لخادم معين على موبايله الشخصي، محتاج:
- Firebase Cloud Messaging (FCM)
- Cloud Function تبعت الـ push notification
- تخزين FCM token لكل خادم

ده تطوير مستقبلي ممكن يتضاف لاحقاً.

## طلب الصلاحية من المستخدم
التطبيق بيطلب صلاحية الإشعارات تلقائياً من `ScheduleProvider`.
لو المستخدم رفض، الإشعارات مش هتشتغل، لكن التوزيع هيتسجل عادي.
